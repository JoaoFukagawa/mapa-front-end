RELATÓRIO TÉCNICO – DESENVOLVIMENTO FRONT-END

1. Introdução

O presente projeto foi desenvolvido como parte da atividade MAPA da disciplina de Programação Front-End da Unicesumar. O objetivo foi criar um website completo e responsivo utilizando HTML5, CSS3 e JavaScript/jQuery, aplicando os conceitos estudados ao longo da disciplina.

O tema escolhido foi um portfólio pessoal de um desenvolvedor front-end fictício chamado João Silva. A escolha do tema se deve à sua versatilidade, pois permite demonstrar habilidades técnicas, organização de projetos, e oferece uma experiência de navegação completa com múltiplas páginas e interatividade.

2. Arquitetura do Site

Quantidade de Páginas

O site é composto por 4 páginas interligadas:

  • index.html (Página Inicial) – Apresentação com seção hero, habilidades e chamada para ação.
  • sobre.html (Sobre) – Biografia do desenvolvedor, habilidades técnicas com barras de progresso animadas e linha do tempo da trajetória profissional.
  • projetos.html (Projetos) – Portfólio visual com cards de projetos e sistema de filtro por categoria.
  • contato.html (Contato) – Formulário de contato com validação via JavaScript e informações de contato.

Navegação

A navegação entre as páginas é feita por meio de um menu fixo no topo (navbar), presente em todas as páginas. O menu destaca a página ativa com um indicador visual (sublinhado animado). Em dispositivos móveis, o menu se transforma em um menu hamburguer com animação de slide lateral.

Layout

O layout foi organizado utilizando conceitos de Flexbox e CSS Grid, garantindo uma distribuição equilibrada dos elementos. Cada página segue uma estrutura semântica com as tags <header>, <nav>, <main>, <section> e <footer>, promovendo acessibilidade e SEO.

3. Tecnologias Utilizadas

HTML5

Utilizado para estruturar todo o conteúdo do site com tags semânticas (header, nav, main, section, footer), meta tags para SEO e caráter responsivo, e formulário com campos tipados (email, tel, select, textarea).

CSS3

Responsável pela estilização completa do site, incluindo:

  • Design moderno com paleta de cores roxa (#6c63ff) e tons neutros.
  • Layout responsivo com media queries para tablets (768px) e celulares (480px).
  • Animações CSS (transitions, keyframes para hover e fade-in).
  • Variáveis CSS (:root) para padronização de cores e espaçamentos.
  • Scrollbar personalizada e botão de voltar ao topo.

JavaScript / jQuery

Foram implementados os seguintes recursos interativos com jQuery e JavaScript puro:

  • Menu mobile: toggle do menu hamburguer com animação de slide.
  • Scroll to top: botão flutuante que aparece ao rolar a página.
  • Fade-in na rolagem: elementos surgem suavemente conforme o usuário scrolla.
  • Barras de progresso animadas: animação dispara quando a seção entra na viewport.
  • Filtro de projetos: exibição/ocultação de cards com fading.
  • Validação de formulário: verificação de campos obrigatórios, formato de email e telefone com feedback visual.
  • Efeito hover nos cards: elevação suave ao passar o mouse.

4. Recursos Implementados

Menu de Navegação

Menu fixo no topo com links para todas as páginas. Em resoluções menores, o menu se transforma em um ícone hamburguer que abre um menu lateral com overlay. O link da página atual recebe destaque visual automático.

Responsividade

O layout foi projetado mobile-first, adaptando-se perfeitamente a diferentes tamanhos de tela através de media queries. As grid layouts se reorganizam (de 4 colunas para 2 e depois 1), os textos ajustam tamanhos, e o menu se adapta para dispositivos touch.

Formulário com Validação

O formulário de contato valida todos os campos via jQuery antes do envio simulado. São verificados: nome (obrigatório, mínimo 3 caracteres), e-mail (obrigatório, formato válido), telefone (formato opcional, mas validado se preenchido), assunto (seleção obrigatória) e mensagem (obrigatória, mínimo 10 caracteres). Campos inválidos recebem borda vermelha e mensagem de erro.

Recursos Interativos (jQuery)

Dois efeitos interativos principais com jQuery:

  • Filtro de Projetos: o usuário pode filtrar os projetos por categoria (Sites, Landing Pages, Aplicações) com animação fadeIn/fadeOut.
  • Animação de Barras de Habilidade: as barras de progresso na página "Sobre" são preenchidas automaticamente quando o usuário rola até a seção.

SEO Básico

Foram aplicadas diversas práticas de SEO:

  • Meta tags description, keywords e author em todas as páginas.
  • Open Graph (og:title, og:description, og:type) para compartilhamento em redes sociais.
  • Tags semânticas HTML5 para melhor interpretação por crawlers.
  • Atributos aria-label para acessibilidade.
  • Hierarquia correta de headings (h1, h2, h3).
  • Favicon personalizado.

5. Desafios e Soluções

Dificuldades Encontradas

  • Menu responsivo: implementar um menu hamburguer funcional que funcionasse bem em mobile sem quebrar o layout em desktop foi desafiador. A solução foi usar JavaScript para controlar classes CSS que alternam entre os estados.
  • Animação ao scroll: fazer com que os elementos aparecessem suavemente ao rolar a página exigiu calcular corretamente as posições dos elementos em relação à viewport.
  • Validação de formulário: criar validações robustas com feedback visual claro para o usuário demandou atenção aos detalhes e testes de diferentes cenários.
  • Filtro de projetos: sincronizar a animação de fade com a reorganização dos cards no grid foi um desafio de timing.

Soluções Adotadas

  • Para o menu, foi utilizado jQuery para controlar o toggle de classe e animar o slide lateral com CSS transitions.
  • Para as animações de scroll, foi implementada uma função que verifica a posição de cada elemento e adiciona a classe "visible" quando apropriado.
  • Para o formulário, cada campo foi validado individualmente com expressões regulares e feedback visual via classes CSS (error/success).
  • Para o filtro, foi usado o método fadeIn() do jQuery com duração controlada para um efeito suave.

7. Considerações Finais

Aprendizados

O desenvolvimento deste projeto permitiu aplicar na prática os conceitos fundamentais de programação front-end. Foi possível compreender como HTML, CSS e JavaScript trabalham em conjunto para criar uma experiência web completa. Especialmente relevante foi o aprendizado sobre:

  • A importância do design responsivo e como implementá-lo com media queries.
  • Como o jQuery simplifica a manipulação do DOM e a criação de efeitos interativos.
  • Técnicas de validação de formulários no lado do cliente.
  • Boas práticas de SEO e acessibilidade.
  • Organização de código e separação de responsabilidades (HTML, CSS, JS).

Melhorias Futuras

Caso o projeto fosse continuado, as seguintes melhorias poderiam ser implementadas:

  • Integração com back-end para envio real de formulários via PHP ou Node.js.
  • Animações mais complexas com CSS Keyframes ou bibliotecas como AOS.js.
  • Modo escuro (dark mode) com toggle via JavaScript.
  • Integração com CMS para gerenciamento dinâmico dos projetos.
  • Otimização de performance com lazy loading de imagens.
  • Testes de compatibilidade cross-browser mais abrangentes.
